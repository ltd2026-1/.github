## Visão Geral

**BizAnalyticsAI** é uma plataforma SaaS de nível empresarial construída para empreendedores, analistas de negócios e empresas em fase de crescimento que precisam de inteligência operacional e preditiva aprofundada. A plataforma combina um poderoso dashboard web com um ecossistema de oito produtos complementares — aplicações desktop, apps mobile, extensões de IDE e bibliotecas para desenvolvedores — todos integrados por meio de um backend Supabase compartilhado com segurança Row Level Security completa.

A plataforma está disponível em [bizanalyticsai.com.br](https://bizanalyticsai.com.br) e serve três portais distintos: o dashboard principal do usuário (`/dashboard`), um painel de controle administrativo (`/admin`) e um portal de colaboração para estudantes (`/alunos`).

### Diferenciais

| Funcionalidade | Descrição |
|---|---|
| **Análise com IA** | Simulações Monte Carlo, modelagem de Churn/LTV e previsão preditiva de receita |
| **Construtor de Plano de Negócios** | Criação de plano de negócios estruturado assistido por IA com exportação para PDF |
| **Motor de Viabilidade Financeira** | Análise de ponto de equilíbrio, período de retorno e calculadoras de margem operacional |
| **Inteligência de Mercado** | TAM/SAM/SOM, segmentação de setor, 5 Forças de Porter e mapeamento competitivo |
| **Gestão de KPIs** | Rastreamento de indicadores-chave em tempo real com definição de metas e benchmarks |
| **Ecossistema Multi-Produto** | 8 produtos complementares: desktop, mobile, extensões de IDE, extensão de navegador, biblioteca Python |
| **White-Label por Usuário** | Cada usuário personaliza seu dashboard — cores, marca, ordem de navegação — com isolamento total |
| **Arquitetura Tri-Portal** | Portais separados e independentemente seguros para usuários, admins e alunos |
| **Multilíngue** | Suporte nativo para Português (pt-BR), Inglês (en-US) e Espanhol (es-ES) |
| **PWA Completo** | Capacidade offline completa com service worker, manifesto e prompt de instalação |
| **OpenAPI** | Spec OpenAPI gerada automaticamente em `/api/v1/openapi` |

### Arquitetura de Autenticação

Três tipos de usuário, cada um com seu próprio contexto de sessão e portal protegido:

```
Tipo de Usuário   Portal         Método de Auth       Verificação de Papel
──────────────────────────────────────────────────────────────────────────
Usuário Regular   /dashboard     JWT cookie Supabase  RLS por usuário
Admin             /admin         JWT cookie Supabase  função is_admin()
Aluno             /alunos        JWT cookie Supabase  registro de aluno
```

**Persistência de sessão:** `localStorage` armazena a identidade do usuário por chave (`bizanalytics_auth_user`) para re-renderização instantânea entre navegações de rotas. O Supabase valida de forma assíncrona. Limpeza no logout previne vazamento de dados entre contas.

### Arquitetura de Personalização do Dashboard

```
Montar → getUser() do Supabase
       ↓
resetDom()                      ← limpa variáveis CSS do usuário anterior
       ↓
Ler localStorage[userId]        ← renderização inicial rápida
       ↓
Carregar de user_dashboard_settings (Supabase)  ← fonte da verdade
       ↓
applyToDom(settings)            ← sobrescreve variáveis CSS no :root
       ↓
--accent · --background · --sidebar · --card · --foreground · --radius
```

# Executar 10.000 cenários Monte Carlo

## Internacionalização

A plataforma inclui suporte multilíngue completo em todas as strings voltadas ao usuário.

| Locale | Idioma | Padrão |
|---|---|---|
| `pt-BR` | Português (Brasil) | ✅ |
| `en-US` | Inglês (Estados Unidos) | — |
| `es-ES` | Espanhol (Espanha) | — |

### Ordem de Detecção

```
1. Cookie NEXT_LOCALE       ← definido pelo LanguageSwitcher, persiste 1 ano
2. Header Accept-Language   ← preferência do navegador
3. Padrão: pt-BR
```

Sem prefixos de URL — o locale é transparente para o usuário. O componente `LanguageSwitcher` oferece duas variantes de exibição:
- `variant="navbar"` — flag completa + label para navegação desktop
- `variant="compact"` — apenas ícone para header do dashboard e mobile

### Cobertura

Todas as chaves de tradução ficam em `messages/pt-BR.json`, `messages/en-US.json` e `messages/es-ES.json`:

- Labels de navegação e títulos de página
- Nomes e descrições dos módulos do dashboard
- Labels de formulários, placeholders e mensagens de validação

## Personalização White-Label

Cada usuário pode personalizar independentemente a aparência do seu dashboard. As configurações são armazenadas em `user_dashboard_settings` no Supabase e são **completamente isoladas por conta** — mudanças feitas por um usuário nunca afetam outro.

### Opções de Customização

| Aba | Opções |
|---|---|
| **Identidade** | Nome da plataforma (substitui "BizAnalyticsAI"), URL do logo |
| **Cores** | Cor de destaque, cor primária, fundo, sidebar, cards, texto |
| **Navegação** | Ordem dos itens da sidebar (↑↓), mostrar/ocultar itens individuais |
| **Visual** | Raio de borda (Reto / Arredondado / Pill), 6 paletas de cores predefinidas |

### Pré-visualização ao Vivo

A página de configurações inclui um painel de pré-visualização ao vivo (coluna direita, sticky) que reflete todas as mudanças em tempo real — sidebar simulada, card e botão — antes de o usuário clicar em "Salvar alterações".

### Implementação Técnica

```typescript
// Chave de cache por usuário — previne vazamento de configurações entre contas
const cacheKey = (userId: string) =>
  `bizanalytics_dash_settings_${userId}`;

// Aplicado às CSS custom properties do :root
applyToDom(settings);
// Mapeia para: --accent · --ring · --background · --sidebar · --card
//              --foreground · --radius

// No logout: variáveis CSS são removidas do :root
resetDom();

// No login de um usuário diferente: resetDom() antes de aplicar novas configurações
```

### Garantias de Isolamento

- `useState` sempre inicializa com `DEFAULT_SETTINGS` — nunca lê localStorage antes de conhecer o user ID
- Chave de cache no localStorage inclui o user ID
- `resetDom()` chamado em todo login para limpar variáveis CSS do usuário anterior
- RLS do Supabase garante que as linhas de `user_dashboard_settings` sejam acessíveis apenas pelo seu dono
- Logout limpa variáveis CSS do `:root`

## Deploy

### Vercel (Recomendado)

Conecte o repositório GitHub ao Vercel para deploys automáticos em produção a cada push para `main`.

```bash
# Configuração única via CLI
npm i -g vercel
vercel --prod
```

**Configuração Vercel:**
- Preset de framework: **Next.js** (detectado automaticamente)
- Versão do Node.js: **20.x**
- Adicione todas as variáveis de ambiente em **Configurações do Projeto → Variáveis de Ambiente**
- Configure os ambientes **Produção** e **Preview** separadamente

### Supabase

1. Crie um projeto em [supabase.com](https://supabase.com)
2. Abra o **SQL Editor** e execute cada migração em `supabase/migrations/` em ordem
3. Habilite **Row Level Security** em todas as tabelas (as migrações cuidam disso)
4. Em **Authentication → Configuração de URL**, adicione seu domínio Vercel às URLs de Redirect
5. Copie sua **URL do Projeto** e **anon key** para `.env.local`
